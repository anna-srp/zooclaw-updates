- remove Playwright `serial` mode from nine production E2E suites whose scenarios can establish independent state
- refactor model-switching to use the test-scoped authenticated `page` fixture instead of one shared page
- start a fresh chat session before each worker-scoped chat scenario and make file-preview reopening generate its own file card

This prevents one real product failure from marking the rest of an unrelated suite as did-not-run, so the production report preserves the full failure inventory.
