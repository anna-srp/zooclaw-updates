## Summary

- add one generic dismissible-overlay DOM contract and replace popup-specific E2E handlers
- update stale workspace routing, anonymous-access, agent lifecycle, i18n, theme, and keyboard assertions
- use the stable Soulmate agent for recovery scenarios instead of the removed Agent Studio entry

## Root cause

The latest scheduled production run had 26 failures and one flaky test. The largest group was caused by the Gift Paywall overlay intercepting pointer events. Other failures came from application contracts that had changed without the E2E expectations following them: installed-agent chats now navigate by `workspace_id`, anonymous chat access redirects to the landing page, app navigation is locale-free, and dark-mode surfaces use gradients rather than a fixed background color.

Popup dismissal also depended on a growing list of feature-specific selectors. Dismissible overlays now expose `data-e2e-overlay="dismissible"` and `data-e2e-overlay-dismiss`; shared `DialogContent` applies the contract automatically. Required-action overlays remain outside this contract.
