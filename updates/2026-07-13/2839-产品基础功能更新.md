claw-interface 对 KB 保持零感知——只在生命周期节点向 proxy 报告事件，授权判定/存储/执行全在 proxy：

- `kb_grant_sync_service`：安装/卸载事件 best-effort（3s
超时、永不失败安装、`KB_GRANT_ADMIN_KEY` 未配置 = 特性整体关闭）；发布绑定注册 mandatory（拒绝即发布失败）
- 安装/卸载收尾接线：卸载前按 (uid, pack) 检查其他计算机上的活跃安装，避免误撤（评审 F3）
- 发布链：`kb_ref` 搭 `listing_metadata` 便车，Pack 生效前注册绑定；拒绝 →
`pack.kb_binding_refused`
- `/admin/cron/kb-grant-reconcile`：按 (uid, pack) 分组、全量分页扫描（无截断盲区，评审
F4），丢失事件/死机器安装收敛；`docs/cron-triggers.md` 同步更新
- BFF：`GET/POST /knowledge-base/kbs` 转发；`POST
/knowledge-base/kbs/{id}/editors` —— owner
按邮箱/手机号**精确匹配**添加编辑者（防枚举统一报错），解析 uid 后转发 proxy 显式授权端点（评审 F1 后的 v2.0
交接模型）
- web 上传页：库选择下拉 + 内联新建（乐观合并防 refetch 竞态），默认"不入库"= 现状行为逐字节不变
- Pack 模型、安装/分享主流程**零改动**

评审修复已含（对抗性评审 2026-07-13）：元数据触发的 editor 授权链删除（不可验证出处 =
自助提权）、多机聚合、对账全量分页、事件短超时、tsc 修复等——详见 spec 第 2/7 节。
