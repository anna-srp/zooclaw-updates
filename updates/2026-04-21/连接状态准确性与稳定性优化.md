# 连接状态显示更准确，修复新用户启动报错

## 核心宣传点

修复了多个影响使用体验的连接相关问题：连接状态现在需要 WebSocket 和 Mattermost 同时在线才显示为"已连接"，防止误报；聊天界面不再夹杂内部心跳检测消息；新用户首次使用时不再出现钱包报错。

## 原始内容

fix(web): require both WS+MM for connection status + anti-jitter debounce (#1090)
Previously connection status could show 'connected' even when only one of WebSocket or Mattermost was connected. Now requires both to be live. Added anti-jitter debounce to prevent rapid state flickering.

fix(web): filter HEARTBEAT status summaries from chat (#1087)
HEARTBEAT status messages were appearing in chat, polluting the conversation. Now filtered out before rendering.

fix(eca-501): skip new-user wallet probe + retry bootstrap on 502/504 (#1086)
New users were getting errors because wallet probe was called before wallet was created. Now skipped for new users. Bootstrap retries on 502/504 for better resilience.
