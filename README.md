# ZooClaw 产品更新库

这个仓库记录了 ZooClaw 每天的产品更新，每天自动同步一次。

你可以直接用自然语言问 Agent，让它帮你从这里查信息，比如：

> "最近有哪些新功能上线？"
> "上周有什么高优先级的更新？"
> "vibe-drama Agent 最新版本有什么变化？"
> "帮我整理一下本周的更新，写成用户公告"
> "这周有哪些内容适合发到 Discord？"

---

## 📁 这里有什么

```
zooclaw-updates/
├── updates/          ← 每条更新一个文件，按日期分文件夹
│   ├── 2026-04-21/
│   │   ├── vibe-drama-Agent-升级-v1.0.9.md
│   │   └── Mattermost附件内联预览.md
│   └── ...
├── raw/              ← 原始 GitHub commit 数据（给 agent 用，人读不友好）
├── index.md          ← 所有更新的总目录（Agent 查信息的入口）
├── changelog.md      ← 人类可读的更新日志，带 emoji 分类
└── templates/        ← 文件模板
```

**建议从 `index.md` 开始**——它列出了所有更新的标题、类型、日期，Agent 可以先看这个，再按需打开具体文件。

---

## 📋 每条更新包含什么

每个更新文件长这样：

```
---
title: "vibe-drama Agent 升级至 v1.0.9"
type: "Agent 上架/更新"
priority: "高"
date: "2026-04-21"
status: "待审核"
channels: ""
---

# vibe-drama Agent 升级至 v1.0.9

## 核心宣传点
一句话说清楚这个更新对用户有什么价值

## 原始内容
对应的 GitHub commit 原文
```

| 字段 | 含义 |
|------|------|
| `type` | Bug Fix / 新功能上线 / Skill 上架/更新 / Agent 上架/更新 / 产品基础功能更新 / 其他公告 |
| `priority` | 高（重要功能/新 Agent）/ 中（用户可感知的改进）/ 低（小优化） |
| `status` | 待审核（未发布）/ 已发布 / 内部-跳过（内部技术变更，不对外） |
| `channels` | 计划发布的渠道，如 Discord、站内弹窗、社媒 |
| 核心宣传点 | 面向用户的一句话描述，可以直接用于对外发布 |

---

## 🔄 更新频率

每天北京时间 **09:00** 自动同步前一天的 GitHub commits，过滤掉纯内部技术变更，只保留用户能感知到的更新。

---

## 💡 给 Agent 的使用提示

如果你是 Agent，读取这个仓库时建议：

1. **先读 `index.md`** — 快速定位日期和标题
2. **按需打开具体文件** — 获取核心宣传点和原始内容
3. **`status: 内部-跳过` 的条目跳过** — 这些是内部技术变更，不对外
4. **`raw/` 目录按需读取** — 包含完整的原始 commit 数据，适合需要技术细节时使用
5. **`changelog.md` 适合生成周报/月报** — 已按时间倒序排列，带 emoji 分类标签
